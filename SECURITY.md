# Security Policy

## Supported Version

Security fixes are applied to the latest revision of the default branch.

## Reporting a Vulnerability

Do not open a public issue for an undisclosed vulnerability.

Use GitHub's private vulnerability reporting:

https://github.com/0xsalome/sound-scanner-oss/security/advisories/new

Private vulnerability reporting must be enabled in the public repository's security settings after the repository is made public.

Include reproduction steps, affected browser and operating-system versions, and the expected impact. Do not attach personal recordings or credentials.

## Privacy Boundary

Sound Scanner processes camera, microphone, and recording data in the browser. The project does not provide a server-side media upload or storage service.
